Projet réalisation d'une base de données et des KPI

Durant ce projet, nous devions réaliser une base de données pour une entreprise et réaliser des KPI grâce à ces chiffres. Nous avions comme base des dossiers avec différents types de fichiers (Access, CSV, Excel, txt). Nous avons donc fait des traitements afin de pouvoir les transformer en fichiers csv pour pouvoir les réutiliser facilement avec Talend. Cette transformation se faisait via Python. Ensuite, Talend prenait les fichiers CSV et les organisait en tables de données que nous connections à une base de données Oracle. Nous vérifions l’ajout de données à l’aide de Visual Studio Code. Une fois que nous avions la certitude que toutes les tables étaient bien remplies, nous avions utilisé Power BI pour nous connecter à la base de données et faire ressortir des KPI pertinents.

Projet création de dockers

Durant ce projet que nous avons réalisé avant les vacances d’hiver, il nous était demandé de générer un site web à l’aide d’une API que nous générions grâce à plusieurs tables de données. Pour cela, nous avons coupé en plusieurs parties le travail. Tout d’abord, nous avons réalisé un premier docker qui lançait un programme Python qui générait une base de données et qui y implémentait les données disponibles dans les différentes tables qui étaient fournies. Après cela, nous avions un autre docker avec la base de données et un autre docker qui venait récupérer les données afin d’en faire des API. Enfin, un dernier Docker venait créer un site web afin de permettre à l’utilisateur de se balader à travers les données que nous avions retransmis grâce à l’API. Cela montrait toute une arborescence qui était complexe à comprendre.

Projet Big Data

Ce projet avait pour but de nous faire comprendre l'étendue des possibilités qui s'offrent à nous avec le Big Data. Nous avons donc pu faire un projet qui se basait tout d'abord sur la récupération de tweets (fictifs). L'objectif derrière était surtout d'être capable de récupérer en communication continue les données des tweets afin de pouvoir faire un traitement statistique dessus dans un second temps en créant un modèle afin d'estimer si les tweets étaient plutôt à caractère positif ou négatif. Cela m'a permis de voir différentes notions du Big Data, comme la récupération de données en continu, le stockage de ces données et le traitement de ces données à l'aide d'algorithmes de machine learning.
